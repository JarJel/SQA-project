### 3.2 Kebutuhan Antarmuka (Interface Requirements)

Kebutuhan antarmuka mendefinisikan titik interaksi antara aplikasi **Sapopoe** dengan entitas eksternal. Spesifikasi ini mencakup antarmuka pengguna, antarmuka perangkat keras, antarmuka perangkat lunak, dan antarmuka komunikasi untuk memastikan interoperabilitas sistem yang lancar.

#### 3.2.1 Antarmuka Pengguna (User Interface)
Desain antarmuka pengguna Sapopoe mengusung konsep *Modern Wealth Management* dengan fokus pada kemudahan navigasi dan visualisasi data yang bersih.
* **Gaya Desain:** Menggunakan tema *Dark Mode* sebagai standar dengan aksen warna emas (#D4AF37) untuk memberikan kesan eksklusif dan profesional.
* **Responsivitas:** Antarmuka harus bersifat *fully responsive* menggunakan *Tailwind CSS*, sehingga mampu beradaptasi secara otomatis pada layar Desktop (1920x1080), Tablet, maupun Mobile (360x800).
* **Komponen Interaktif:**
    - *Dashboard* dengan widget ringkasan saldo yang menonjol.
    - *Form* input transaksi yang ringkas untuk mempercepat pencatatan harian.
    - Visualisasi grafik menggunakan library *Chart.js* atau *Recharts* yang interaktif (dapat disentuh/klik untuk melihat detail angka).
    - Fitur *Image Cropper* pada pengaturan profil untuk memastikan foto pengguna selalu tampil proposional.

#### 3.2.2 Antarmuka Perangkat Keras (Hardware Interface)
Sebagai aplikasi berbasis web, Sapopoe tidak memerlukan perangkat keras khusus, namun membutuhkan spesifikasi minimum untuk performa optimal:
* **Perangkat Input:** Mendukung *Keyboard*, *Mouse/Touchpad*, dan Layar Sentuh (*Touchscreen*).
* **Resolusi Layar:** Minimum 360px (lebar) untuk perangkat *mobile* agar seluruh elemen UI tidak terpotong.
* **Konektivitas:** Perangkat harus memiliki modul jaringan (WiFi/Seluler) untuk melakukan pertukaran data dengan server secara *real-time*.

#### 3.2.3 Antarmuka Perangkat Lunak (Software Interface)
Aplikasi Sapopoe berinteraksi dengan beberapa komponen perangkat lunak berikut:
* **Web Browser:** Kompatibel dengan peramban berbasis *Chromium* (Google Chrome, Microsoft Edge, Opera), Safari, dan Mozilla Firefox versi terbaru.
* **Sistem Operasi:** *Cross-platform* (Windows, macOS, Linux, Android, iOS) selama menjalankan peramban yang didukung.
* **Server-Side Environment:** - **PHP 8.2+** sebagai bahasa pemrograman *backend*.
    - **MySQL 8.0** sebagai sistem manajemen basis data.
* **Layanan Pihak Ketiga:**
    - **SMTP Service (Gmail API):** Digunakan untuk mengirimkan email otomatis (OTP dan notifikasi keamanan).

#### 3.2.4 Antarmuka Komunikasi (Communication Interface)
Sistem menggunakan standar protokol komunikasi modern untuk menjamin keamanan data:
* **Protokol HTTP/HTTPS:** Seluruh komunikasi antara *Frontend* (React.js) dan *Backend* (Laravel) dilakukan melalui protokol terenkripsi HTTPS.
* **RESTful API:** Pertukaran data menggunakan standar arsitektur REST dengan format **JSON** (*JavaScript Object Notation*).
* **API Authentication:** Menggunakan **Laravel Sanctum** untuk mengelola *Bearer Token* yang dikirimkan pada setiap *header* permintaan dari sisi klien.
* **CORS (Cross-Origin Resource Sharing):** Konfigurasi keamanan pada server untuk memastikan hanya domain resmi aplikasi yang diizinkan untuk memanggil API.
