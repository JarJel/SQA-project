# 3.6 Desain Antarmuka (User Interface)

Bagian ini mendokumentasikan rancangan antarmuka pengguna (*User Interface*) aplikasi **Sapopoe** secara visual dan deskriptif. Setiap halaman dirancang mengikuti prinsip **konsistensi visual**, **kemudahan navigasi**, dan **kejelasan informasi** untuk mendukung pengalaman manajemen keuangan pribadi yang nyaman dan intuitif.

---

## Ringkasan Halaman

| No | Nama Halaman | Jalur Akses | Deskripsi Singkat |
|---|---|---|---|
| 1 | Halaman Login | `/login` | Pintu masuk utama — autentikasi pengguna terdaftar |
| 2 | Halaman Daftar | `/register` | Pendaftaran akun baru dengan validasi real-time |
| 3 | Halaman Input OTP | `/verify-otp` | Verifikasi identitas melalui kode OTP via email |
| 4 | Notifikasi OTP | *(popup/toast)* | Konfirmasi pengiriman email OTP |
| 5 | Setup Dompet | `/setup-wallet` | Inisialisasi dompet pertama sebelum masuk Dashboard |
| 6 | Dashboard | `/dashboard` | Pusat kendali keuangan — ringkasan, grafik, transaksi |
| 7 | Input Transaksi | `/transactions/create` | Formulir pencatatan pemasukan dan pengeluaran |
| 8 | Halaman Portofolio | `/portfolio` | Ikhtisar aset, hutang, tabungan, dan anggaran |
| 9 | Halaman Pengaturan | `/settings` | Manajemen profil, akun, dan preferensi aplikasi |

## 1. Halaman Login

![Halaman Login](https://github.com/user-attachments/assets/ce3fd5fc-f5bf-4247-8a9f-2c5f57b059f7)

### Deskripsi

Halaman Login merupakan titik masuk utama aplikasi Sapopoe. Halaman ini menampilkan form autentikasi dengan desain yang bersih dan terfokus, menempatkan elemen utama di tengah layar untuk memaksimalkan fokus pengguna.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Logo Sapopoe | Gambar / Teks | Identitas brand di bagian atas form |
| Input Email | `<input type="email">` | Kolom email dengan validasi format |
| Input Password | `<input type="password">` | Kolom password dengan toggle visibilitas |
| Tombol Login | `<button>` — *Primary* | Aksi submit form autentikasi |
| Tautan Daftar | `<a>` | Navigasi ke halaman pendaftaran akun baru |
| Tautan Lupa Password | `<a>` | Navigasi ke alur reset password via OTP |

### Alur Interaksi

```
Pengguna mengisi email dan password
        ↓
Klik tombol "Login"
        ↓
[Gagal] → Pesan error ditampilkan inline di bawah field
        ↓
[Berhasil] → Redirect ke /dashboard
```

## 2. Halaman Daftar

![Halaman Daftar](https://github.com/user-attachments/assets/c7168039-d3ce-473f-a82f-6eacfaf0da8e)

### Deskripsi

Halaman Daftar menyediakan formulir pendaftaran akun baru. Form menerapkan validasi *real-time* pada setiap field sehingga pengguna mendapat umpan balik langsung sebelum submit. Desain mengikuti pola yang konsisten dengan halaman Login untuk menjaga kohesi visual.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Input Nama Lengkap | `<input type="text">` | Validasi minimal 3 karakter |
| Input Email | `<input type="email">` | Validasi format dan keunikan email |
| Input Password | `<input type="password">` | Validasi minimal 8 karakter |
| Input Konfirmasi Password | `<input type="password">` | Harus cocok dengan field password |
| Tombol Daftar | `<button>` — *Primary* | Submit form pendaftaran |
| Tautan Login | `<a>` | Navigasi kembali ke halaman Login |

### State: Form Setelah Diisi

![Halaman Daftar Terisi](https://github.com/user-attachments/assets/fa160704-f0dd-48be-af46-80976775116a)

Ketika semua field terisi dengan valid, tombol **Daftar** aktif dan siap di-submit. Indikator validasi hijau muncul pada setiap field yang lolos validasi.

### Alur Interaksi

```
Pengguna mengisi semua field
        ↓
Validasi real-time berjalan di setiap field
        ↓
Klik tombol "Daftar"
        ↓
[Email sudah terdaftar] → Pesan error inline
        ↓
[Berhasil] → Backend kirim OTP ke email
           → Redirect ke /verify-otp
```

## 3. Halaman Input OTP

![Halaman Input OTP](https://github.com/user-attachments/assets/625ae911-92ea-460f-9e95-92150c449926)

### Deskripsi

Halaman ini ditampilkan setelah proses pendaftaran berhasil dan sistem telah mengirimkan kode OTP ke email pengguna. Antarmuka dirancang sesederhana mungkin agar pengguna fokus pada satu aksi: memasukkan kode verifikasi.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Informasi Email Tujuan | Teks statis | Menampilkan sebagian alamat email tujuan OTP |
| Input Kode OTP | `<input>` 6 digit | Input numerik, umumnya terdiri dari 6 kotak terpisah |
| Tombol Verifikasi | `<button>` — *Primary* | Submit kode OTP untuk divalidasi backend |
| Tautan Kirim Ulang | `<a>` / `<button>` | Memicu pengiriman ulang OTP dengan *cooldown timer* |
| Timer Kedaluwarsa | Teks dinamis | Countdown masa berlaku kode OTP |

### Alur Interaksi

```
Pengguna membuka email dan menyalin kode OTP
        ↓
Memasukkan 6 digit kode pada field input
        ↓
Klik "Verifikasi"
        ↓
[Kode salah / expired] → Pesan error, opsi kirim ulang aktif
        ↓
[Kode valid] → Status user diubah ke active
             → Bearer Token diterbitkan
             → Redirect ke /setup-wallet
```

## 5. Setup Dompet Pertama

![Setup Dompet](https://github.com/user-attachments/assets/561f2e89-5a81-4496-8214-2bd6b4ba143a)

### Deskripsi

Halaman ini adalah *onboarding step* wajib yang muncul **hanya sekali** — setelah verifikasi OTP berhasil dan sebelum pengguna pertama kali mengakses Dashboard. Sistem tidak mengizinkan akses Dashboard hingga minimal satu dompet didaftarkan. Desain halaman menggunakan pendekatan *wizard* yang ramah untuk pengguna baru.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Progress Indicator | Stepper visual | Menunjukkan posisi langkah dalam alur onboarding |
| Input Nama Dompet | `<input type="text">` | Contoh: Kas, BCA, GoPay |
| Dropdown Tipe Dompet | `<select>` | Pilihan: Tunai, Bank, E-Wallet |
| Input Saldo Awal | `<input type="number">` | Saldo awal yang dimiliki pada dompet ini |
| Tombol Simpan & Mulai | `<button>` — *Primary* | Submit data dompet dan masuk ke Dashboard |
| Tautan Lewati | `<a>` | Opsional — melewati setup (tidak direkomendasikan) |

### Alur Interaksi

```
Pengguna mengisi nama, tipe, dan saldo awal dompet
        ↓
Klik "Simpan & Mulai"
        ↓
Backend menyimpan data ke tabel wallets
        ↓
Redirect ke /dashboard (akses pertama kali)
```

## 6. Dashboard

![Dashboard](https://github.com/user-attachments/assets/f6109822-53eb-4760-8115-64a7166d83b2)

### Deskripsi

Dashboard adalah halaman utama dan pusat kendali aplikasi Sapopoe. Semua informasi keuangan penting tersaji dalam satu tampilan: ringkasan saldo, arus kas, grafik pengeluaran per kategori, dan riwayat transaksi terkini. Pengguna dapat mengambil keputusan keuangan harian cukup dari halaman ini.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Header Salam | Teks dinamis | Menampilkan nama pengguna dan tanggal hari ini |
| Kartu Total Saldo | *Card* komponen | Agregat saldo dari semua dompet aktif |
| Kartu Pemasukan | *Card* komponen | Total pemasukan bulan berjalan |
| Kartu Pengeluaran | *Card* komponen | Total pengeluaran bulan berjalan |
| Grafik Arus Kas | *Line / Bar Chart* | Visualisasi pemasukan vs pengeluaran per periode |
| Grafik Kategori | *Pie / Donut Chart* | Distribusi pengeluaran per kategori |
| Daftar Transaksi Terkini | *List* komponen | 5–10 transaksi terakhir dengan detail ringkas |
| Tombol Tambah Transaksi | FAB / `<button>` | Shortcut cepat ke halaman input transaksi |
| Navigasi Bawah | *Bottom Navigation* | Akses cepat ke Dashboard, Transaksi, Portofolio, Pengaturan |

### Struktur Layout

```
┌─────────────────────────────────────┐
│  Header: Salam + Nama Pengguna      │
├──────────┬──────────┬───────────────┤
│  Total   │ Masuk    │   Keluar      │
│  Saldo   │ Bulan Ini│   Bulan Ini   │
├──────────┴──────────┴───────────────┤
│       Grafik Arus Kas               │
├─────────────────────────────────────┤
│  Grafik Kategori  │  Ringkasan      │
├─────────────────────────────────────┤
│       Transaksi Terkini             │
│  [Tambah Transaksi]                 │

```

## 7. Halaman Input Transaksi

![Input Transaksi](https://github.com/user-attachments/assets/1a2dd95d-0e60-41a1-a977-e887395e2a19)

### Deskripsi

Halaman ini adalah formulir pencatatan transaksi keuangan harian. Pengguna dapat mencatat pemasukan (*income*) maupun pengeluaran (*expense*) dengan memilih dompet, kategori, dan mengisi nominal serta deskripsi. Sistem secara otomatis memvalidasi kecukupan saldo sebelum menyimpan data.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Toggle Tipe | Tab / Toggle | Pilihan: **Pemasukan** atau **Pengeluaran** |
| Input Nominal | `<input type="number">` | Nilai transaksi dalam Rupiah |
| Dropdown Dompet | `<select>` | Pilih dompet sumber/tujuan dana |
| Dropdown Kategori | `<select>` | Pilih kategori transaksi (dinamis berdasarkan tipe) |
| Input Tanggal | `<input type="date">` | Default ke hari ini, dapat diubah |
| Input Deskripsi | `<textarea>` | Catatan opsional untuk transaksi |
| Dropdown Hutang | `<select>` — Opsional | Menghubungkan transaksi ke cicilan hutang tertentu |
| Tombol Simpan | `<button>` — *Primary* | Submit transaksi ke backend |
| Tombol Batal | `<button>` — *Secondary* | Kembali ke halaman sebelumnya |

### Alur Validasi Backend

```
Form disubmit
        ↓
Backend cek: Saldo Dompet >= Nominal?
        ↓
[Tidak] → 400 Bad Request
        → Pesan error: "Saldo tidak mencukupi"
        ↓
[Ya] → Atomic DB Transaction:
       1. INSERT ke tabel transactions
       2. UPDATE saldo di tabel wallets
       3. Cek limit budget kategori
        ↓
[Overbudget] → 201 + flag is_overbudget: true
             → Toast notifikasi peringatan merah
        ↓
[Normal] → 201 Created
         → Update grafik Dashboard real-time
```

## 9. Halaman Pengaturan

![Pengaturan](https://github.com/user-attachments/assets/8bac87f0-2375-47fb-b628-a159e5456202)

### Deskripsi

Halaman Pengaturan adalah pusat manajemen akun dan preferensi personal pengguna. Di sini pengguna dapat memperbarui data profil, mengganti foto avatar (yang akan diproses otomatis ke format WebP), mengubah password, serta mengelola data-data pendukung seperti kategori dan dompet.

### Elemen Antarmuka

| Elemen | Tipe | Keterangan |
|---|---|---|
| Avatar Pengguna | Gambar + Tombol Ubah | Menampilkan foto profil saat ini; klik untuk unggah foto baru |
| Nama Lengkap | `<input type="text">` | Dapat diedit langsung |
| Alamat Email | Teks statis | Tidak dapat diubah (terikat ke akun OTP) |
| Tombol Ganti Password | `<button>` | Membuka form ganti password |
| Manajemen Kategori | *List* + Tambah/Hapus | Kelola kategori transaksi custom |
| Manajemen Dompet | *List* + Tambah/Edit | Kelola daftar dompet dan akun keuangan |
| Tombol Simpan Profil | `<button>` — *Primary* | Menyimpan perubahan data profil |
| Tombol Keluar | `<button>` — *Danger* | Logout dari sesi aplikasi |

### Alur Upload Avatar

```
Pengguna klik foto profil / tombol "Ubah Foto"
        ↓
File picker terbuka
        ↓
Pengguna pilih file gambar (JPG / PNG / WebP)
        ↓
Frontend kirim POST /api/profile/upload (multipart/form-data)
        ↓
Backend: Crop otomatis (center) → Encode ke .webp
        ↓
Simpan ke storage/app/public/avatars/
        ↓
UPDATE avatar_url di tabel users
        ↓
Foto profil baru tampil tanpa reload halaman
```

## Konsistensi Desain Antarmuka

### Palet Warna

| Peran | Warna | Penggunaan |
|---|---|---|
| *Primary* | Hijau / Biru teal | Tombol utama, aksen brand, indikator positif |
| *Danger* | Merah | Pesan error, tombol hapus, notifikasi overbudget |
| *Warning* | Oranye / Kuning | Notifikasi peringatan batas budget |
| *Success* | Hijau cerah | Konfirmasi sukses, status aktif |
| *Neutral* | Abu-abu | Teks sekunder, border, background card |

### Komponen Berulang

| Komponen | Digunakan Di |
|---|---|
| *Bottom Navigation Bar* | Dashboard, Transaksi, Portofolio, Pengaturan |
| *Toast Notification* | Konfirmasi aksi sukses, peringatan overbudget, error |
| *Card* | Dashboard (ringkasan), Portofolio (dompet, hutang, tabungan) |
| *Inline Error* | Semua form input (Login, Daftar, Transaksi) |
| *Loading Spinner* | Setiap operasi async ke backend |

### Prinsip UX yang Diterapkan

| Prinsip | Implementasi |
|---|---|
| **Feedback Langsung** | Validasi form real-time, toast notifikasi setelah setiap aksi |
| **Pencegahan Error** | Tombol submit dinonaktifkan hingga form valid |
| **Konsistensi** | Pola navigasi, warna, dan tipografi seragam di semua halaman |
| **Efisiensi** | FAB tambah transaksi selalu tersedia di Dashboard |
| **Keamanan** | Session berakhir otomatis; data terproteksi per `user_id` |
