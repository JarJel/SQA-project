
### 3.5.2 Activity Diagram

*Activity Diagram* ini menggambarkan alur kerja fungsionalitas sistem secara mendalam, mulai dari proses autentikasi hingga logika pemrosesan transaksi keuangan pada aplikasi **Sapopoe**.

#### A. Alur Registrasi, Autentikasi, dan Inisialisasi Akun
Diagram ini menjelaskan bagaimana sistem menangani verifikasi OTP dan kewajiban pembuatan dompet pertama sebelum masuk ke Dashboard.

```mermaid
activityDiagram
    start
    :User Melakukan Registrasi Akun;
    :Sistem Mengirim Kode OTP ke Email;
    repeat
      :User Memasukkan Kode OTP;
    backward:Kode Salah;
    repeat while (Apakah OTP Valid?) is (Tidak)
    ->Ya;
    :Akun Terverifikasi;
    if (Apakah Sudah Memiliki Dompet?) then (Ya)
      :Masuk ke Dashboard;
    else (Tidak)
      :Tampilkan Form Inisialisasi Dompet;
      :User Mengisi Saldo Awal;
      :Simpan Data Dompet;
      :Masuk ke Dashboard;
    endif
    stop
