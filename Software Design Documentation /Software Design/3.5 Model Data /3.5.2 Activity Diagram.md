### 3.5.2 Activity Diagram



*Activity Diagram* ini menggambarkan aliran kerja fungsional dan logika operasional di dalam sistem **Sapopoe**. Diagram dibagi menjadi dua bagian utama untuk memberikan detail yang jelas pada setiap proses bisnis inti.



#### A. Alur Registrasi dan Inisialisasi Akun

Fokus pada validasi identitas melalui OTP dan kewajiban pengaturan aset awal sebelum pengguna dapat menggunakan fitur utama.



```mermaid

flowchart TD

    %% Simbol Start & End

    Start(((Start)))

    End(((End)))



    Start --> Reg[User Melakukan Registrasi]

    Reg --> OTP[Sistem Kirim Kode OTP ke Email]

    

    %% Loop Verifikasi OTP

    OTP --> InputOTP[User Input Kode OTP]

    InputOTP --> DecOTP{OTP Valid?}

    DecOTP -- Tidak --> InputOTP

    DecOTP -- Ya --> Verify[Akun Terverifikasi]

    

    %% Logika Wajib Dompet

    Verify --> DecWallet{Memiliki Dompet?}

    DecWallet -- Tidak --> FormWallet[Tampilkan Form Buat Dompet]

    FormWallet --> InitBal[User Isi Saldo Awal]

    InitBal --> SaveWallet[Simpan Data Dompet]

    SaveWallet --> Dash[Masuk ke Dashboard]

    

    DecWallet -- Ya --> Dash

    Dash --> End



flowchart TD

    Start(((Start))) --> Action[User Memilih Aksi]

    

    Action --> DecAction{Jenis Aksi?}

    

    %% Alur Update Foto

    DecAction -- Update Foto --> Up1[Upload Foto Profil]

    Up1 --> Up2[Kirim ke Backend Laravel]

    Up2 --> Up3[Proses Crop & Konversi ke WebP]

    Up3 --> Up4[Simpan ke Storage & Database]

    Up4 --> Up5[Tampilkan Foto Baru di UI]

    Up5 --> End(((End)))



    %% Alur Catat Pengeluaran

    DecAction -- Catat Pengeluaran --> Exp1[Input Nominal & Kategori]

    Exp1 --> DecBal{Saldo >= Nominal?}

    

    DecBal -- Tidak --> ErrBal[Tampilkan Error: Saldo Tidak Cukup]

    ErrBal --> Cancel[Batalkan Transaksi]

    Cancel --> End

    

    DecBal -- Ya --> Pay[Potong Saldo Dompet Terkait]

    Pay --> SaveExp[Simpan Riwayat Transaksi]

    SaveExp --> DecBud{Melebihi Budget?}

    DecBud -- Ya --> Notif[Berikan Notifikasi Merah/Peringatan]

    Notif --> UpdateGraph[Update Grafik Dashboard Real-time]

    DecBud -- Tidak --> UpdateGraph

    UpdateGraph --> End



    %% Alur Transfer

    DecAction -- Transfer --> Trf1[Pilih Dompet Asal & Tujuan]

    Trf1 --> Trf2[Input Nominal Transfer]

    Trf2 --> DecTrf{Saldo Asal Cukup?}

    DecTrf -- Ya --> Trf3[Potong Saldo Asal & Tambah Saldo Tujuan]

    Trf3 --> TrfLog[Simpan Log Mutasi Transfer]

    TrfLog --> End

    DecTrf -- Tidak --> ErrBal
