### 3.5.2 Activity Diagram

*Activity Diagram* ini menggambarkan aliran kerja fungsional dan logika operasional di dalam sistem **Sapopoe**. Diagram dibagi menjadi dua bagian utama untuk memberikan detail yang jelas pada setiap proses bisnis inti.

#### A. Alur Registrasi dan Inisialisasi Akun
Fokus pada validasi identitas melalui OTP dan kewajiban pengaturan aset awal sebelum pengguna dapat menggunakan fitur utama.

```mermaid
stateDiagram-v2
    %% Titik Hitam Start UML
    [*] --> Registrasi_Akun
    
    Registrasi_Akun --> Kirim_OTP_Email
    Kirim_OTP_Email --> Input_OTP
    
    %% Belah Ketupat Decision UML
    state Cek_OTP <<choice>>
    Input_OTP --> Cek_OTP
    
    Cek_OTP --> Input_OTP : OTP Salah
    Cek_OTP --> Akun_Terverifikasi : OTP Valid
    
    %% Belah Ketupat Decision UML
    state Punya_Dompet <<choice>>
    Akun_Terverifikasi --> Punya_Dompet
    
    Punya_Dompet --> Tampil_Form_Dompet : Belum Ada
    Tampil_Form_Dompet --> Isi_Saldo_Awal
    Isi_Saldo_Awal --> Simpan_Data_Dompet
    Simpan_Data_Dompet --> Masuk_Dashboard
    
    Punya_Dompet --> Masuk_Dashboard : Sudah Ada
    
    %% Bullseye End UML
    Masuk_Dashboard --> [*]
