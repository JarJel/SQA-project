### 2.3 Karakteristik User (User Characteristics)

Sistem **Sapopoe** dirancang menggunakan arsitektur *Single-Role*, di mana aplikasi ini secara eksklusif didedikasikan untuk pengguna akhir (*End-User*). Sistem ini bersifat tertutup dan mandiri, sehingga tidak memiliki entitas *System Administrator*.

#### Diagram Pemetaan Interaksi Pengguna (User Interaction Map)

```mermaid
flowchart TD
    %% Aktor (Menggunakan bentuk lingkaran/oval)
    U1((Personal User))
    U2((Household Manager))

    %% Sistem Sapopoe
    subgraph SAPOPOE_SYSTEM [SISTEM SAPOPOE]
        A[Dashboard & Analytics]
        B[Transaction & Wallet]
        C[Budget & Relasi]
        D[Profile & Security]
    end

    %% Relasi Interaksi
    U1 -.->|Mencatat Harian| B
    U1 -.->|Cek Hutang| C
    U1 -->|Kelola Profil| D

    U2 ==>|Analisis Laporan| A
    U2 ==>|Set Anggaran| C
    U2 -->|Kelola Profil| D

    %% Styling Dasar
    classDef actor fill:#f59e0b,stroke:#0f172a,stroke-width:2px,color:#000,font-weight:bold;
    classDef system fill:#f8fafc,stroke:#333,stroke-width:2px;
    class U1,U2 actor;
    class A,B,C,D system;
