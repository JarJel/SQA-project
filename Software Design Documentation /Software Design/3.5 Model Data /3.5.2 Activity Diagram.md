### 3.5.2 Activity Diagram

*Activity Diagram* ini menggambarkan aliran kerja fungsional dan logika operasional di dalam sistem **Sapopoe**. Diagram dibagi menjadi dua bagian utama untuk memberikan detail yang jelas pada setiap proses bisnis inti.

#### A. Alur Registrasi dan Inisialisasi Akun

Fokus pada validasi identitas melalui OTP dan kewajiban pengaturan aset awal sebelum pengguna dapat menggunakan fitur utama.

```mermaid
flowchart TD
    %% Simbol Start & End
    StartA(((Start)))
    EndA(((End)))

    StartA --> Reg[User Melakukan Registrasi]
    Reg --> OTP[Sistem Kirim Kode OTP ke Email]
    
    %% Loop Verifikasi OTP
    OTP --> InputOTP[User Input Kode OTP]
    InputOTP --> DecOTP{OTP Valid?}
    DecOTP -- Tidak --> InputOTP
    DecOTP -- Ya --> Verify[Akun Terverifikasi]
    
    %% Logika Wajib Dompet
    Verify --> DecWallet{Memiliki Dompet?}
    DecWallet -- Tidak --> FormWallet[Tampilkan Form Buat Dompet]
    FormWallet --> InitBal[User Isi Saldo Awal]
    InitBal --> SaveWallet[Simpan Data Dompet]
    SaveWallet --> Dash[Masuk ke Dashboard]
    
    DecWallet -- Ya --> Dash
    Dash --> EndA
