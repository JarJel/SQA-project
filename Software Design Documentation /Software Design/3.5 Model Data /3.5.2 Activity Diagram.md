### 3.5.2 Activity Diagram

*Activity Diagram* ini menggambarkan aliran kerja fungsional dan logika operasional sistem **Sapopoe**. Diagram ini menggunakan konsep *Swimlane* untuk memisahkan tanggung jawab antara aktivitas yang dilakukan oleh **Pengguna (Frontend)** dan proses yang berjalan di dalam **Sistem (Backend Laravel)**.

#### A. Alur Registrasi dan Inisialisasi Akun
Menggambarkan proses pendaftaran, verifikasi OTP, hingga pembuatan dompet pertama.

```mermaid
flowchart TD
    %% Styling Bentuk sesuai Gambar Referensi (UML)
    classDef startNode fill:#000,stroke:#000,stroke-width:2px,color:#fff;
    classDef endNode fill:#fff,stroke:#000,stroke-width:4px;
    classDef activity fill:#ffffff,stroke:#333,stroke-width:1.5px;
    
    %% SWIMLANE PENGGUNA
    subgraph Pengguna [Pengguna (Aplikasi Client)]
        direction TB
        Start(( )):::startNode
        Reg(Melakukan Registrasi Akun):::activity
        InpOTP(Input Kode OTP):::activity
        InpDompet(Input Saldo Awal Dompet):::activity
        Dash(Akses Halaman Dashboard):::activity
        End((( ))):::endNode
    end
    
    %% SWIMLANE SISTEM
    subgraph Sistem [Sistem (Backend Laravel)]
        direction TB
        Kirim(Kirim OTP ke Email Gmail):::activity
        Verif(Melakukan Verifikasi OTP):::activity
        CekOTP{OTP Valid?}
        CekDompet{Sudah Punya Dompet?}
        SimpanDompet(Simpan Data Dompet):::activity
    end
    
    %% ALUR PROSES
    Start --> Reg
    Reg --> Kirim
    Kirim --> InpOTP
    InpOTP --> Verif
    Verif --> CekOTP
    
    CekOTP -- Tidak --> InpOTP
    CekOTP -- Ya --> CekDompet
    
    CekDompet -- Belum --> InpDompet
    InpDompet --> SimpanDompet
    SimpanDompet --> Dash
    
    CekDompet -- Sudah --> Dash
    Dash --> End
