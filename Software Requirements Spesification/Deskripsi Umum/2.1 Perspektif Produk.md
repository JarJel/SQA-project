### **2.1 Perspektif Produk**

Bagian ini mendefinisikan keberadaan produk dalam lingkungan perangkat lunak dan perangkat keras secara menyeluruh.

**A. Hubungan Sistem (*System Interfaces*)**
Aplikasi SAPOPOE merupakan produk mandiri (*stand-alone*) yang berinteraksi dengan beberapa komponen utama untuk menjalankan fungsinya:
* **Antarmuka Pengguna**: Aplikasi diakses melalui *Web Browser* (seperti Google Chrome atau Microsoft Edge) sebagai jembatan interaksi antara pengguna dan sistem.
* **Antarmuka Basis Data**: Sistem berkomunikasi dengan DBMS MySQL untuk melakukan operasi penyimpanan, pembaruan, dan pengambilan data transaksi keuangan pengguna.
* **Antarmuka Perangkat Lunak**: Aplikasi berjalan di atas *web server* Apache dan dikembangkan menggunakan bahasa pemrograman PHP dengan framework Laravel.

**B. Antarmuka Perangkat Keras (*Hardware Interfaces*)**
Produk ini tidak memiliki perangkat keras khusus, namun memiliki kebutuhan perangkat keras minimal untuk operasional:
* **Sisi Server**: Memerlukan komputer dengan kapasitas penyimpanan dan memori yang cukup untuk menampung database serta menjalankan logika aplikasi.
* **Sisi Pengguna**: Memerlukan perangkat (PC, Laptop, atau Smartphone) yang mampu menjalankan peramban web modern.

**C. Batasan Operasional dan Lingkungan**
Aplikasi ini beroperasi dalam lingkungan *client-server*. Perspektif utamanya adalah menyediakan validasi input yang ketat pada sisi klien sebelum data diproses oleh server untuk menjaga integritas data keuangan (seperti nominal yang tidak boleh nol).

**D. Ketergantungan Produk**
Produk ini sangat bergantung pada:
* Ketersediaan layanan database MySQL; jika database mati, maka fungsi utama aplikasi (pencatatan transaksi) tidak dapat dijalankan.
* Kepatuhan input pengguna terhadap batasan data yang telah ditentukan, seperti limit karakter pada nama relasi atau batasan nominal.

### **2.5 Karakteristik User**

Bagian ini mendefinisikan kelompok pengguna yang akan menggunakan aplikasi SAPOPOE serta tugas-tugas yang akan dilakukan.

| Kategori Pengguna | Deskripsi Tugas | Tingkat Keahlian |
| :--- | :--- | :--- |
| User (Masyarakat Umum) | Melakukan pencatatan transaksi harian, mengelola anggaran bulanan, memantau hutang/piutang, dan mengatur target impian finansial. | Menengah (Mampu mengoperasikan perangkat smartphone/PC dan mengerti navigasi dasar web browser). |
| Administrator (Pengembang) | Melakukan pemeliharaan sistem, mengelola database `t_master`, dan memastikan keamanan akses layanan server. | Ahli (Memahami struktur database, framework Laravel, dan manajemen server). |

**Karakteristik Umum Pengguna:**
* **Edukasi**: Pengguna diharapkan memiliki pemahaman dasar mengenai literasi keuangan, seperti perbedaan antara pemasukan, pengeluaran, dan aset.
* **Kemampuan Teknis**: Pengguna harus familiar dengan antarmuka formulir digital, seperti pengisian data pada kolom **Nominal Pokok** dan penggunaan menu *dropdown*.
* **Kebutuhan Informasi**: Pengguna memerlukan informasi yang transparan mengenai status saldo dompet dan progres tabungan secara *real-time*.
