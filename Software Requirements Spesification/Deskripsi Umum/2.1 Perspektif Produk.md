### **2.1 Perspektif Produk**

[cite_start]Bagian ini mendefinisikan keberadaan produk dalam lingkungan perangkat lunak dan perangkat keras secara menyeluruh[cite: 85].

#### **A. Hubungan Sistem (System Interfaces)**
[cite_start]Aplikasi **SAPOPOE** merupakan produk mandiri (*stand-alone*) yang berinteraksi dengan beberapa komponen utama untuk menjalankan fungsinya[cite: 86]:
* [cite_start]**Antarmuka Pengguna**: Aplikasi diakses melalui *Web Browser* (seperti Google Chrome atau Microsoft Edge) sebagai jembatan interaksi antara pengguna dan sistem[cite: 147].
* [cite_start]**Antarmuka Basis Data**: Sistem berkomunikasi dengan DBMS **MySQL** untuk melakukan operasi penyimpanan, pembaruan, dan pengambilan data transaksi keuangan pengguna[cite: 86, 115].
* [cite_start]**Antarmuka Perangkat Lunak**: Aplikasi berjalan di atas *web server* **Apache** dan dikembangkan menggunakan bahasa pemrograman **PHP** dengan framework **Laravel**[cite: 86].

#### **B. Antarmuka Perangkat Keras (Hardware Interfaces)**
Produk ini tidak memiliki perangkat keras khusus, namun memiliki kebutuhan perangkat keras minimal untuk operasional:
* [cite_start]**Sisi Server**: Memerlukan komputer dengan kapasitas penyimpanan dan memori yang cukup untuk menampung database serta menjalankan logika aplikasi[cite: 86].
* [cite_start]**Sisi Pengguna**: Memerlukan perangkat (PC, Laptop, atau Smartphone) yang mampu menjalankan peramban web modern[cite: 147].

#### **C. Batasan Operasional dan Lingkungan**
[cite_start]Aplikasi ini beroperasi dalam lingkungan *client-server*[cite: 85, 131]. [cite_start]Perspektif utamanya adalah menyediakan validasi input yang ketat pada sisi klien sebelum data diproses oleh server untuk menjaga integritas data keuangan (seperti nominal yang tidak boleh nol)[cite: 160].

#### **D. Ketergantungan Produk**
Produk ini sangat bergantung pada:
* [cite_start]Ketersediaan layanan database **MySQL**; jika database mati, maka fungsi utama aplikasi (pencatatan transaksi) tidak dapat dijalankan[cite: 112, 113].
* [cite_start]Kepatuhan input pengguna terhadap batasan data yang telah ditentukan, seperti limit karakter pada nama relasi atau batasan nominal[cite: 102, 107].
