## 3. Bug Report

### 🐛 BUG #001 — Missing Maximum Validation on Transaction Amount

| Atribut | Detail |
|---------|--------|
| **ID Bug** | BUG-001 |
| **Ditemukan pada** | BVA-07 (BB02-BVA07-amount1000000000.json) |
| **Severity** | 🟠 Medium |
| **Priority** | 🔴 High |
| **Modul** | `TransactionController@store` |
| **File** | `app/Http/Controllers/API/TransactionController.php` |
| **Deskripsi** | Sistem menerima `amount = 1.000.000.000` (satu miliar) tanpa menolaknya, padahal nilai tersebut berada di luar batas wajar transaksi keuangan personal |
| **Root Cause** | Rule validasi `'amount' => 'required\|numeric\|min:1'` tidak memiliki constraint `max` |
| **Expected** | HTTP 422 — Validation error |
| **Actual** | HTTP 201 — Transaksi berhasil dibuat, saldo akun bertambah 1 miliar |
| **Dampak** | Nilai ekstrem dapat tersimpan ke database, menyebabkan ketidakakuratan laporan keuangan dan potensi overflow |
| **Rekomendasi** | Tambahkan `max:999999999` pada rule validasi: `'amount' => 'required\|numeric\|min:1\|max:999999999'` |
| **Status** | 🔴 OPEN |

## 4. Kesimpulan

Dari **55 test case** yang dieksekusi:

- ✅ **54 test case PASSED** (98.2%)
- ❌ **1 test case FAILED** (1.8%)
- 🐛 **1 bug ditemukan** (BUG-001 — Missing max validation)

Sistem **Midnight Finance** secara keseluruhan telah memenuhi sebagian besar kriteria pengujian fungsional. Satu kelemahan ditemukan pada validasi batas atas field `amount` yang memerlukan perbaikan segera sebelum deployment ke production.

**Rekomendasi:** Lakukan hotfix pada `TransactionController.php` dengan menambahkan `max:999999999` pada rule validasi `amount`, kemudian jalankan ulang BVA-07 untuk konfirmasi perbaikan.
