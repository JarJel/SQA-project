## 4.3 Analisis Kegagalan (Defect Analysis)

Berdasarkan hasil di atas, ditemukan 3 temuan kritis yang perlu segera diperbaiki:

1.  **Bug TC-CART-03 (Severity: Medium):** Sistem tidak memvalidasi tipe input pada kolom jumlah pesanan. Hal ini dapat menyebabkan error *NaN (Not a Number)* pada kalkulasi total harga di backend.
2.  **Bug TC-PAY-04 (Severity: High):** Kegagalan penanganan *callback* saat user membatalkan pembayaran secara paksa. Sistem seharusnya mempertahankan status 'Pending' bukan langsung mengubah ke 'Fail'.
3.  **Bug TC-ADM-04 (Severity: Low):** Kurangnya validasi ukuran file gambar menyebabkan tampilan katalog menu menjadi tidak rapi dan lambat dimuat.

## 4.5 Kesimpulan & Rekomendasi Akhir

### Kesimpulan
Secara keseluruhan, **Sistem Pemesanan Café Berbasis Web** telah menunjukkan performa yang stabil pada alur bisnis utama (*Happy Path*). Meskipun terdapat beberapa temuan bug, sistem tidak mengalami *crash* total selama pengujian dilakukan. Tingkat keberhasilan 85% menunjukkan bahwa aplikasi berada pada jalur yang benar menuju kesiapan produksi.

### Rekomendasi
1.  **Perbaikan Prioritas:** Tim pengembang wajib memperbaiki bug **TC-PAY-04** sebelum sistem digunakan untuk transaksi uang asli.
2.  **Enhancement:** Disarankan untuk menambahkan fitur *auto-refresh* pada dashboard admin agar pesanan baru dapat terlihat tanpa perlu memuat ulang halaman secara manual.
3.  **Retesting:** Setelah bug diperbaiki, harus dilakukan *Regression Testing* pada modul keranjang dan pembayaran untuk memastikan tidak ada fitur lain yang rusak akibat perbaikan tersebut.
