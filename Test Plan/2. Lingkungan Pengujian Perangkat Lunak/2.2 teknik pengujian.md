# 2.2 Teknik Pengujian

Pengujian Midnight Finance dilakukan dengan tiga pendekatan berlapis sesuai taksonomi Software Testing:

### 2.2.1 White Box Testing (Berbasis Kode Internal)

Penguji memeriksa logika internal *source code* backend Laravel. Teknik yang digunakan:

| Teknik | Modul Target |
|:---|:---|
| **Desk Checking** | Penelusuran manual logika kalkulasi saldo transaksi |
| **Code Walkthrough** | Review kolaboratif alur autentikasi (AuthController) |
| **Formal Inspection** | Pemeriksaan kelengkapan & format validasi input transaksi |
| **Control Flow Testing** | Pengujian cabang (branch) pada CategoryController@store |
| **Basis Path Testing** | Hitung Cyclomatic Complexity & jalur independen pada Login + Transaksi |
| **Data Flow Testing** | Pengujian aliran variabel saldo pada Transfer Antar Portofolio |
| **Loop Testing** | Pengujian perulangan pada logika zero-filling 7 hari Analytics |

### 2.2.2 Black Box Testing (Berbasis Fungsionalitas)

Penguji memvalidasi sistem dari sudut pandang pengguna tanpa melihat kode internal. Teknik yang digunakan:

| Teknik | Modul Target |
|:---|:---|
| **Boundary Value Analysis** | Field `amount` pada POST /api/transactions |
| **Equivalence Partitioning** | Form registrasi (name, email, password) |
| **Decision Table Testing** | Kombinasi kondisi login |
| **Comparison Testing** | Konsistensi respons API antar endpoint |
| **Sample Testing** | Sampling dataset riwayat transaksi |
| **Robustness Testing** | Input di luar spesifikasi pada form login |
| **Behaviour Testing** | User journey: Registrasi → Login → Catat Transaksi |
| **Performance Testing** | Kecepatan respons endpoint dashboard & analytics |
| **Endurance Testing** | Stabilitas sistem saat endpoint diakses berulang kali |
| **Cause-Effect Testing** | Analisis root cause: bug saldo tidak terupdate |

### 2.2.3 Gray Box Testing (Kombinasi)

Penguji memiliki pengetahuan parsial tentang struktur internal. Teknik yang digunakan:

| Teknik | Modul Target |
|:---|:---|
| **Orthogonal Array Testing** | Kombinasi filter & sort pada GET /api/transactions |
| **Matrix Testing** | Matriks kombinasi parameter filter transaksi |
| **Regression Testing** | Sinkronisasi saldo otomatis setelah CRUD transaksi |
| **Pattern Testing** | Eksplorasi edge case Dashboard & Analytics |
