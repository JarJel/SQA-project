# 2.2 Teknik Pengujian (Testing Methodologies)

Pengujian sistem dilakukan dengan pendekatan berlapis untuk menjamin kualitas dari sisi kode internal hingga antarmuka akhir:

### 2.2.1 Black Box Testing (Functional Focused)
Teknik ini merupakan pilar utama dalam pengujian ini. Tester akan memposisikan diri sebagai pengguna akhir yang tidak mengetahui struktur kode program. Fokus pengujian adalah:
* **Graph-Based Testing:** Memetakan alur navigasi dari halaman beranda hingga transaksi selesai.
* **Equivalence Partitioning:** Membagi data input menjadi bagian-bagian yang valid dan tidak valid untuk melihat respon sistem.

### 2.2.2 Boundary Value Analysis (Limit Testing)
Fokus pada pengujian nilai-nilai ekstrim di batas sistem. Contoh nyata:
* Menguji input jumlah item tepat pada angka 1, angka maksimal stok, dan angka di luar batas (0 atau di atas stok).
* Menguji panjang karakter pada nama menu atau catatan pesanan pelanggan.

### 2.2.3 End-to-End (E2E) Testing
Mensimulasikan satu siklus hidup pesanan secara utuh:
1. Pelanggan melakukan registrasi.
2. Login dan memilih menu.
3. Menambahkan ke keranjang dan melakukan penyesuaian jumlah.
4. Checkout dengan memilih nomor meja.
5. Melakukan pembayaran via payment gateway.
6. Admin menerima notifikasi dan mengubah status menjadi 'Diproses'.
7. Sistem mengirimkan invoice digital kepada pelanggan.

### 2.2.4 User Acceptance Testing (UAT)
Melibatkan calon pengguna nyata (pemilik café atau staf) untuk memberikan validasi apakah alur yang dibangun sudah sesuai dengan kenyataan operasional di lapangan. UAT memastikan aplikasi tidak hanya bebas bug, tetapi juga "layak pakai" secara bisnis.
