# 2.1 Material Pengujian (Detailed Test Materials)

Untuk menjamin validitas hasil pengujian dan memastikan bahwa sistem dapat berjalan stabil di berbagai kondisi penggunaan nyata, operasional pengujian pada **Sistem Pemesanan Café Berbasis Web** didukung oleh infrastruktur dan material yang dikonfigurasi secara spesifik. Berikut adalah rincian mendalam mengenai material pengujian yang digunakan:

### 2.1.1 Perangkat Keras (Hardware Infrastructure)
Pengujian dilakukan pada berbagai platform fisik untuk mensimulasikan penggunaan oleh pelanggan (mobile-first) dan pengelola café (back-office):

* **Workstation Administrator & Developer:**
    * **Unit:** Laptop/PC Desktop dengan spesifikasi minimum Processor Quad-Core (Intel Core i5 Gen-11 atau AMD Ryzen 5 setara).
    * **Memory:** RAM 8GB/16GB DDR4 untuk menangani proses *multitasking* antara browser, database manager, dan tools pengujian.
    * **Display:** Monitor dengan resolusi Full HD (1920x1080) untuk memastikan seluruh dashboard admin tampil tanpa adanya elemen UI yang tumpang tindih.
* **Perangkat Seluler Pelanggan (Mobile Devices):**
    * **Android Representative:** Perangkat dengan layar 6+ inch, RAM 4GB, menjalankan sistem operasi Android 10 atau di atasnya. Fokus pada pengujian *Chrome Mobile*.
    * **iOS Representative:** iPhone dengan layar Retina, menjalankan iOS 15 atau terbaru, untuk memverifikasi performa pada browser *Safari*.
    * **Low-End Device Simulation:** Menggunakan perangkat dengan spesifikasi lebih rendah untuk memastikan optimasi gambar dan kecepatan pemuatan halaman menu tetap terjaga.

### 2.1.2 Perangkat Lunak & Sistem Operasi (Software Environment)
Sistem diuji pada berbagai ekosistem perangkat lunak untuk meminimalisir masalah kompatibilitas:

* **Sistem Operasi:** Windows 10/11, macOS (Sonoma/Ventura), Linux (Ubuntu 22.04) untuk sisi server/admin, serta Android dan iOS untuk sisi klien.
* **Web Browser (Cross-Browser Testing):**
    * **Google Chrome (Engine V8):** Sebagai browser utama dengan pangsa pasar terbesar.
    * **Mozilla Firefox:** Untuk memastikan kepatuhan terhadap standar web engine Gecko.
    * **Safari:** Fokus pada optimasi render CSS dan elemen transisi pada perangkat Apple.
    * **Microsoft Edge:** Verifikasi kompatibilitas pada browser berbasis Chromium lainnya.
* **Database Management:** Environment staging menggunakan MySQL 8.0 atau PostgreSQL yang dikonfigurasi identik dengan database produksi untuk menguji integritas kueri.

### 2.1.3 Alat Bantu Pengujian (Testing Tools & Utilities)
Penggunaan perangkat lunak khusus untuk mengotomatisasi dan mempermudah proses identifikasi bug:

* **Postman / Insomnia:** Digunakan untuk melakukan *API Testing* pada setiap endpoint (seperti proses POST pesanan atau GET katalog menu) guna memastikan respon JSON sesuai dengan kontrak API.
* **Chrome DevTools:** Alat utama untuk melakukan *Mobile Emulation*, memantau *Network Latency*, mendeteksi kebocoran memori pada JavaScript, serta menguji responsivitas elemen UI (*Media Queries*).
* **Lighthouse:** Digunakan untuk mengaudit performa, aksesibilitas, dan *Best Practices* dari aplikasi web, khususnya pada kecepatan pemuatan aset gambar menu.
* **JMeter (Optional):** Untuk melakukan pengujian beban ringan (*Simulated Concurrent Users*) saat proses checkout berlangsung serentak.

### 2.1.4 Data Pengujian (Test Data)
Material pengujian juga mencakup kumpulan data buatan (*dummy data*) yang dirancang untuk menguji batas sistem:

* **Dataset Menu:** Terdiri dari 50+ item menu dengan variasi ukuran gambar (dari 100KB hingga 2MB) untuk menguji efisiensi *Lazy Loading*.
* **Akun Pengguna:** Kumpulan akun dengan berbagai role (Admin, Kasir, Pelanggan Terdaftar, Pelanggan Tamu/Guest).
* **Skenario Input:** Data teks dengan karakter spesial, string panjang (untuk menguji *word-wrap*), serta skenario angka nol/negatif pada kolom jumlah pesanan.
* **Payment Sandbox:** Akun pengujian dari *Payment Gateway* (seperti Midtrans/Xendit sandbox) untuk mensimulasikan status pembayaran 'Pending', 'Success', dan 'Expired'.
