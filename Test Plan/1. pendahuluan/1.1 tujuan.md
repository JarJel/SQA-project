# 1.1 Tujuan Pengujian

Dokumen *Software Test Plan* ini disusun sebagai pedoman utama bagi tim penjamin kualitas dalam melaksanakan rangkaian pengujian pada **Sistem Manajemen Keuangan Pribadi Midnight Finance**. Sasaran yang ingin dicapai melalui proses pengujian ini meliputi:

### 1.1.1 Validasi Fungsionalitas Sistem

Menjamin bahwa setiap fitur yang dikembangkan berfungsi sesuai dengan *Software Requirement Specification* (SRS). Hal ini mencakup:

- **Verifikasi Autentikasi:** Memastikan sistem registrasi, verifikasi OTP via email, dan login menggunakan Laravel Sanctum berjalan aman dan benar untuk semua skenario (sukses, email tidak terdaftar, password salah, belum verifikasi).
- **Akurasi Pencatatan Transaksi:** Memastikan setiap pencatatan transaksi income dan expense mencatat histori dengan benar dan memperbarui saldo portofolio secara otomatis dan akurat.
- **Validasi Manajemen Kategori:** Menjamin fitur tambah, edit, dan hapus kategori bekerja sesuai spesifikasi, termasuk pencegahan duplikat kategori per user.
- **Kebenaran Dashboard & Analytics:** Memastikan ringkasan saldo, statistik bulan ini, serta grafik 7 hari terakhir menampilkan data yang akurat termasuk logika *zero-filling* untuk hari tanpa transaksi.

### 1.1.2 Integritas dan Konsistensi Data

Mengingat aplikasi mengelola data keuangan pribadi yang sensitif, pengujian bertujuan untuk:

- **Sinkronisasi Saldo Real-Time:** Memastikan saldo portofolio (`FinancialAccount`) diperbarui secara instan setiap kali transaksi dicatat, diedit, atau dihapus menggunakan mekanisme `DB::transaction`.
- **Pencegahan Data Bocor Antar User:** Menjamin semua query data difilter berdasarkan `user_id` sehingga tidak ada data satu user yang dapat diakses oleh user lain.
- **Konsistensi Rollback:** Memverifikasi bahwa jika terjadi error saat pencatatan transaksi, mekanisme `DB::rollBack()` memastikan tidak ada perubahan saldo yang tersimpan sebagian.

### 1.1.3 Mitigasi Risiko Keamanan

- **Proteksi Endpoint:** Memverifikasi seluruh endpoint privat terlindungi middleware `auth:sanctum` dan mengembalikan HTTP 401 jika diakses tanpa token.
- **Validasi Input:** Memastikan semua input divalidasi di sisi backend Laravel untuk mencegah data tidak valid masuk ke database.
- **Anti Brute-Force OTP:** Memverifikasi throttle `5 percobaan/menit` pada endpoint `/verify-otp` dan `3 permintaan/menit` pada `/resend-otp` berjalan dengan benar.

### 1.1.4 Pengalaman Pengguna

- **Konsistensi Respons API:** Memastikan seluruh endpoint mengembalikan struktur JSON yang konsisten sehingga frontend React.js dapat mengolah data dengan benar.
- **Pesan Error yang Informatif:** Memastikan sistem mengembalikan pesan error yang jelas dan spesifik untuk setiap skenario kegagalan.
